# Sorriso Curitiba - Clínica Odontológica

## 🎯 Objetivo do Projeto
Este projeto consiste no desenvolvimento de um site institucional para a clínica odontológica fictícia **Sorriso Curitiba**. O objetivo principal é fornecer uma plataforma clara e intuitiva para que os pacientes conheçam a clínica, a equipe profissional, as especialidades atendidas e consigam solicitar um agendamento de consulta de forma prática.

O projeto foi desenvolvido como parte da Avaliação Prática da disciplina de Desenvolvimento Web, aplicando conceitos fundamentais de front-end sem o uso de frameworks externos.

## 🚀 Tecnologias Utilizadas
* **HTML5:** Estruturação semântica de toda a página utilizando as novas tags (`<header>`, `<nav>`, `<main>`, `<section>`, `<footer>`).
* **CSS3:** Estilização visual, uso de posicionamento avançado, organização de layout responsivo através de Media Queries para adaptação em dispositivos móveis.
* **JavaScript (Vanilla JS):** Manipulação dinâmica do DOM, tratamento de eventos do navegador, interceptação do envio de formulário e validação lógica de dados do lado do cliente.

## 📂 Estrutura das Seções (Single Page Application)
O site foi construído no modelo de página única com navegação ancorada, dividindo-se em 4 itens principais:

* **Início (Home):** Boas-vindas ao usuário com uma apresentação de impacto (Hero Section) e chamada para ação (*Call to Action*) para agendamento.
* **Conheça-nos (Sobre):** Seção institucional apresentando a infraestrutura da clínica e do corpo clínico.
* **Especialidades (Serviços):** Listagem dos tratamentos oferecidos (Limpeza, Clareamento, Ortodontia e Implantes).
* **Agendamento (Contato):** Formulário interativo para coleta e validação de dados de novos pacientes.

## 🛠️ Funcionalidades Implementadas
* **Navegação Suave:** Links do menu que rolam suavemente pela tela até a seção desejada através da propriedade `scroll-behavior`.
* **Design Responsivo:** Layout adaptável para telas de computadores, tablets e smartphones usando regras de `@media screen`.
* **Validação de Formulário via JS:** O script valida se o nome possui o tamanho mínimo ideal, limpa strings e valida se o número de telefone informado possui a quantidade correta de dígitos para um padrão nacional com DDD.
* **Simulação de Persistência:** Interceptação do evento `submit` com `preventDefault()`, limpando os campos após validação bem-sucedida e exibindo uma mensagem temporária de sucesso ao usuário (`setTimeout`).
